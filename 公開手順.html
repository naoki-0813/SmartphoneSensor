G-METER を公開してスマホで使う手順

GitHub Pages（ギットハブ ページズ＝GitHubが無料で用意してくれる公開サーバ）を使って、 HTTPSのURLで世界に公開し、スマホのブラウザから接続します。

なぜHTTPSが必要？
このアプリはスマホのモーションセンサーとGPSを使います。これらは安全のため、 HTTPS（鍵マークのつく安全な通信）でしか動きません。 GitHub Pages は自動でHTTPSにしてくれるので、ぴったりです。
公開後にスマホで開くURL（予定） https://naoki-0813.github.io/SmartphoneSensor/g-meter.html

1変更をGitHubに反映（push）する

すでにコミット済みなら、最新の状態がGitHub上にあるか確認します。Claude に 「pushして」と頼むか、自分でやる場合は次のコマンドです。

  1. git push origin main を実行する
  2. ブラウザで github.com/naoki-0813/SmartphoneSensor を開き、g-meter.html が表示されることを確認

2GitHub Pages を有効にする

  1. リポジトリのページ上部の Settings（設定）をクリック
  2. 左メニューの Pages をクリック
  3. Build and deployment の Source を Deploy from a branch に設定
  4. Branch で main ／ フォルダは / (root) を選び、Save を押す
数分すると、ページ上部に 「Your site is live at https://naoki-0813.github.io/SmartphoneSensor/」 と表示されます。これが公開された合図です。

3公開URLをスマホで開く

パソコンのブラウザでまず動作確認してから、スマホで開きます。 アプリ本体は g-meter.html なので、URLの末尾を忘れずに付けます。

スマホで開くURL https://naoki-0813.github.io/SmartphoneSensor/g-meter.html
スマホのカメラでこのQRコードを読み取ると、上のURLが開きます
（QR画像はネット接続時のみ表示されます）

4センサーの利用を許可する

  • アプリの 「スタート」 ボタンを押す
  • iPhone: 「モーションと方向へのアクセスを許可しますか？」が出たら 許可
  • 位置情報（GPS）: 許可を求められたら 許可（速度表示に必要）
  • 停車中に 「ゼロ設定」、直進加速中に 「前方向セット」 を押すと精度が上がります
安全のため、操作は必ず停車中に。運転中は画面を見ないでください。 スマホは車にしっかり固定してから使いましょう。

？うまくいかないとき

  • 404 と出る：URLの末尾 /g-meter.html が抜けていないか確認。 公開直後は反映に数分かかることがあります。
  • センサー値が来ない：必ず https:// で開いていますか？ パソコンや file://（ファイルを直接開く）ではセンサーは動きません。
  • 速度が「0」のまま：GPSの許可と、屋外（空が見える場所）かを確認。 屋内や駐車場では電波が届きにくいです。
  • iPhoneで許可が出ない：Safari の設定で「モーションと画面の向きのアクセス」がオンか確認。
公開URLは一度設定すれば、コードを更新して git push するたびに自動で反映されます。
