# エラー解決ガイドスキル

エラーや不具合を、原因の特定 → 修正 → 再発防止まで、初心者にも分かるように進める。

## 進め方

1. **情報を集める**
   - エラーメッセージ全文（貼られていなければ依頼する）。
   - いつ起きるか（`pnpm dev` 中／ビルド時／特定の操作時）。
   - 直前に何を変えたか。

2. **エラーを読み解く**
   - エラーの**意味**を一言で日本語にする（専門用語は補足する）。
   - メッセージ内のファイル名・行番号を `path:line` で示し、該当箇所を読む。

3. **原因を特定する**
   - よくある原因から確認する:
     - 依存の入れ忘れ → `pnpm install`
     - import のパス/名前ミス
     - 型エラー（TypeScript）
     - null/undefined を触っている
     - 環境変数の未設定
   - 推測で直さず、根本原因にたどり着いてから直す。

4. **直す**
   - 最小の変更で修正する。何を・なぜ変えたかを 1〜2 行で説明する。
   - 修正後、`pnpm lint` / `pnpm test` / `pnpm dev` などで**直ったことを確認**する。
     確認できないものは「未確認」と正直に伝える。

5. **再発防止**
   - 同じエラーを避けるコツを 1 つ添える（例: 「import は補完を使うと打ち間違いが減ります」）。

## 注意
- エラーを隠すだけの対処（握りつぶし・`any` 化・チェック無効化）はしない。
- 直す前に「どこを変えるか」を一言示してから進める。
