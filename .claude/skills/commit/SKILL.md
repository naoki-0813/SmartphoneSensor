# 日本語コミット作成スキル

変更を、規約に沿った日本語コミットとして記録する。

## 手順

1. **変更を把握する**
   - `git status` と `git diff`（必要なら `git diff --staged`）で内容を確認する。
   - 変更が複数の目的に分かれている場合は、分けてコミットすることを提案する。

2. **ブランチを確認する**
   - いま `main` にいて新機能/修正をする場合は、先にブランチを切ることを提案する
     （例: `git checkout -b feat/login-form`）。

3. **メッセージを作る**（Conventional Commits 風・日本語）
   - 1 行目: `種別: 要約`（50 字程度）
     - `feat:` 新機能 / `fix:` バグ修正 / `refactor:` 整理 / `docs:` 文書
     - `chore:` 設定・依存 / `test:` テスト / `style:` 整形のみ
   - 必要なら空行を空けて本文（なぜこの変更をしたか）。

4. **コミットする**
   - `git add` で対象を選び、`git commit` する。
   - 秘密情報（`.env` 等）が含まれていないか確認してから add する。

5. **報告**
   - 何をコミットしたかを日本語で 1〜2 行で伝える。
   - 利用者が望めば `git push` と PR 作成（`gh pr create`）を案内する。

## 注意
- **利用者が頼んだときだけ** commit / push する。勝手に push しない。
- `--no-verify` などフックの回避はしない。フックが失敗したら原因を直す。
- コミットメッセージは日本語。署名や Co-Authored-By は付けない（個人用途のため）。
