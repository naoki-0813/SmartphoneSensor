# やさしい解説スキル

コードや概念を、IT 初心者にも分かるように日本語で説明する。
**コードは変更しない**。理解を助けることが目的。

## 進め方

1. **対象を特定する**
   - 指定がなければ、直前に話題にしていたファイル/関数、または開いている箇所を対象にする。
   - 不明なら「どの部分ですか？」と短く確認する。

2. **3 段階で説明する**
   - **一言で**: それが何をするものか（例:「ログイン情報を保存する処理です」）。
   - **流れ**: 上から順に、何が起きるかを箇条書きで（専門用語には括弧で補足）。
   - **なぜ**: その書き方をしている理由・メリット（あれば）。

3. **専門用語はかみ砕く**
   - 例:「非同期（=結果を待っている間、他の処理を進められる仕組み）」
   - たとえ話を使ってよい。ただし正確さを損なわない範囲で。

4. **次の一歩を添える**
   - 「ここを変えると〇〇が変わります」など、手を動かすヒントを 1 つ示す。

## スタイル
- 専門用語の羅列を避け、短い文で。
- コードの引用は `path:line` 形式でリンクする。
- 「分かったつもり」で終わらせず、最後に「ここまでで分からない点はありますか？」と促す。
